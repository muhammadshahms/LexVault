Auth flow — Supabase + session
1
Register / Login
User submits form → supabase.auth.signUp() or signInWithPassword() → Supabase returns JWT token
2
Session store
JWT stored in express-session → user ID + role saved server-side → cookie set on client
3
Role fetch
On login → query Supabase profiles table → fetch role field → attach to req.session.user
4
Route guard middleware
auth.js checks session exists → roleGuard.js checks req.session.user.role → redirect if unauthorized
5
Dashboard redirect
Admin → /dashboard/admin   Attorney → /dashboard/attorney   Client → /dashboard/client

Build order — 4 phases
Phase 1
Day 1
Project setup + Auth
Init Node.js + Express + EJS. Connect Supabase. Build login/register. Role-based session. Dashboard redirect by role. Middleware setup.
Phase 2
Day 2
Cases module + Deadlines
Case CRUD routes + controllers. Supabase RLS policies. EJS views for cases list/detail/create. Deadline calculation. Cron job for email alerts.
Phase 3
Day 3
Client portal + Admin
Attorney assigns client to case. Client dashboard (read-only). Admin panel — all users, all cases, role management. RLS finalization.
Phase 4
Day 4
Stripe billing + Polish
Stripe checkout session. Webhook handler. Subscription gating. UI polish — sidebar, alerts, responsive. Deploy to IDX / Railway / Render. README + GitHub.
